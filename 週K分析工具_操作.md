## 五、如何解讀報告

### Step 1：看統計摘要

| 指標 | 意義 |
|------|------|
| R:R ≥ 1.5 的檔數 | 值得重點關注（賺賠比至少 1.5:1） |
| 週線多頭檔數 | 價格 > 週MA10 > 週MA20，趨勢向上 |
| 有效時段數 | 當日完整執行了幾個時段（通常5個） |

### Step 2：辨識市場標籤

代號旁出現：
- **深藍色標籤 [上市]** = 台灣證券交易所上市股票（TradingView 連結 `TWSE:代號`）
- **深黃色標籤 [上櫃]** = 台灣證券交易所上櫃股票（TradingView 連結 `TPEX:代號`）

### Step 3：優先看 R:R 高的標的

R:R ≥ 2.0 → 優先研究（上市或上櫃均可）
R:R ≤ 0 → 現價已超過壓力，不建議追高

### Step 4：看來源標籤

```
持倉中 + 09:00 + 13:00  → 多時段出現且在持倉中，高確信度
09:00 + 10:00            → 跨時段出現，訊號持續
09:00 (單一時段)          → 僅一次訊號，謹慎觀察
```

### Step 5：持倉中的進出場參考

報告中持倉股票的「出場目標」可做為當前持倉的**目標設定參考**；「停損」可確認**停損防線是否仍有效**。

### Step 6：R:R 為負數怎麼辦

R:R 為負代表**目標低於進場價**，通常是因為現價已高過壓力，此時不建議進場，已持倉者考慮輕倉或出場。

## 六、常見問題

| 問題 | 解法 |
|------|------|
| 報告只有持倉，沒有選股 | 當天 `hourly/` 無當日資料，選股工具未執行 |
| 法人欄全部顯示「中立」 | `prev_scores.json` 無對應個股，不影響週K計算 |
| 週K資料不足（< 20 週） | 新上市/上櫃股票資料不足，自動略過 |
| Email 沒收到 | 確認 Gmail 應用程式密碼；檢查 `stock_email_config.json` |
| R:R 顯示「—」 | 進場價 ≤ 停損（極少數情況），計算無意義自動略過 |
| 上櫃股票未出現在報告 | 確認選股工具當日有正常執行，且上櫃清單抓取成功 |
| MD 報告沒有上櫃段落 | 當日篩選後上櫃無符合條件股票（R:R < 1.2 或策略不符），屬正常 |
