# 台股選股工具 v2 — 篩選方法與評分說明

> 對應腳本：`tw_stock_screener_v2.py`

---

## 四階段篩選流程

```
全市場 2,000+ 檔（上市 + 上櫃合併掃描）
    │
    ▼ 第一階段：日線掃描
    │  - 排除今日成交量 < 1,000 張（流動性不足）
    │  - 計算日線評分
    │  - 日線分 < 30 → 淘汰
    │
    ▼ 第二階段：月營收基本面（FinMind）
    │  - 抓取近 13 個月營收
    │  - 計算近 3 個月平均 YoY
    │  - 依 YoY 給予加/扣分
    │
    ▼ 第三階段：三大法人
    │  - 上市：TWSE T86 官方 API
    │  - 上櫃：TPEX 三大法人每日買賣超 API
    │  - 抓取最近 5 個交易日外資/投信買賣超
    │  - 計算連續買超天數
    │  - 連續 >= 3 天 → 標記 **
    │
    ▼ 第四階段：60分線精算
       - 對候選股下載 60 天小時 K 線
       - 計算小時線評分
       - 合成綜合分
       - 上市依綜合分排序取前 10 名
       - 上櫃依綜合分排序取前 10 名（各別獨立）
```

## 技術指標評分細則

### 日線設定

| 參數 | 值 |
|------|-----|
| RSI 合理區間 | 25 ~ 60 |
| 量增門檻 | 今日量 > 20日均量 × 1.5 倍 |
| 一階通過門檻 | 日線分 >= 30 |

### 60分線設定

| 參數 | 值 |
|------|-----|
| RSI 合理區間 | 20 ~ 65 |
| 量增門檻 | 今日量 > 20日均量 × 1.8 倍 |

## 評分加減分明細

| 訊號 | 條件 | 加/扣分 |
|------|------|---------|
| **KD 黃金交叉** | K 從下穿越 D 向上 | **+25** |
| **MACD 柱翻正** | MACD 柱狀圖從負轉正 | **+20** |
| **均線多頭排列** | MA5 > MA10 > MA20 | **+15** |
| KD 低檔 | K < 30 且 D < 30 | +10 |
| 站上 MA20 | 收盤 > MA20 | +10 |
| 成交量放大 | 量 > 均量 × 1.5x（日）/ 1.8x（時） | +10 |
| RSI 適中 | RSI 在合理區間內 | +10 |
| 布林下緣反彈 | 前一根跌破下軌，當根回到下軌以上 | +10 |
| MACD 增強 | 柱狀圖持續增加且為正 | +8 |
| 盤整型態支撐 | 見下方說明 | +8 |
| 部分均線多頭 | MA5 > MA20（不完整排列） | +5 |
| RSI 過熱 | RSI > 72 | **-15** |
| 跌破季線 | 收盤 < MA60 | -5 |
| RSI 過弱 | RSI < 25（日）/ < 20（時） | -5 |

> 技術評分下限 0 分，上限 100 分

## 月營收基本面加分

資料來源：FinMind 免費 API（近 3 個月平均 YoY 年增率）

| YoY | 加/扣分 |
|-----|---------|
| > 20% | **+20** |
| > 10% | +15 |
| > 0% | +8 |
| > -10% | +0 |
| <= -10% | -5 |

> 月營收快取以**月為單位**，每月初自動更新，同月內不重複抓取。

---

## 三大法人判斷

抓取**最近 5 個交易日**外資及陸資、投信的每日買賣超（張），上市與上櫃分別抓取後合併。

| 市場 | 資料來源 |
|------|---------|
| 上市 | TWSE T86（官方公開，收盤後更新） |
| 上櫃 | TPEX 三大法人每日買賣超 API（收盤後更新） |

- **連續 >= 3 天買超** → 在報告中標示 `**`（紅色粗體）

| 欄位 | 意義 |
|------|------|
| 外資 +XXX | 今日外資買超 XXX 張 |
| 外資 -XXX | 今日外資賣超 XXX 張 |
| **+XXX** | 連續 >= 3 天買超（高確信度） |

---

## 盤整型態偵測（+8 分）

同時符合以下四個條件才觸發：

1. 過去 60 個交易日內曾出現**大量紅K**（收漲且量 > 均量 2 倍以上）
2. 大量紅K之後，收盤**沒有跌破**大量紅K的最低點（允許 1% 誤差）
3. 整理期間平均成交量 < 大量紅K量的 70%（縮量整理）
4. 目前股價**低於**大量紅K的收盤價（仍在整理中）

報告訊號欄顯示：`支撐$XXX整理N天`，其中 XXX 為支撐價，可作為停損參考。

---

## 綜合分計算公式

| 時段 | 公式 |
|------|------|
| 盤中（09:00–13:30） | `60分線評分 × 60% + 日線評分 × 40% + 月營收分 + 型態分` |
| 盤後 | `日線評分 × 70% + 60分線評分 × 30% + 月營收分 + 型態分` |
| 無 60 分線資料 | `日線評分 + 月營收分 + 型態分` |

> 綜合分上限 100 分

---

## 黃底共振訊號

**日線分 >= 40 且 60分線分 >= 40** → 報告中以黃色底色標示。

代表波段（日線）與短線（60分線）同時出現訊號，可信度更高。

---

## 模擬下單規則

| 設定 | 值 |
|------|-----|
| 進場條件 | 綜合分 >= 60 |
| 每時段最多新增 | **上市 3 檔 + 上櫃 3 檔（各別獨立，合計最多 6 檔）** |
| 最多同時持倉 | 10 檔 |
| 持有期間 | 5 個交易日 |
| 出場方式 | 第 5 個交易日收盤價自動出場 |

5 個時段（09:00 / 10:00 / 11:00 / 12:00 / 13:00）**各自獨立追蹤**，
每個時段進場的倉位分別計算勝率與報酬，方便比較哪個時段進場勝率最高。

> 上市與上櫃的進場名額**互不佔用**：同一時段，上市最多新增 3 檔，上櫃也最多新增 3 檔。
