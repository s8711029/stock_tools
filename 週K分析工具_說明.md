# 台股週K壓力/支撐分析工具 v2 — 說明

> 對應腳本：`tw_stock_weekly_analysis.py`
> 最後更新：2026-04-12

---

## 這個工具在做什麼？

每天收盤後（18:00）**自動讀取當日所有時段的選股結果與持倉中股票**，
對每一檔進行週K技術分析，計算壓力/支撐區間，給出進場價、出場目標與停損建議。

```
讀取今日 hourly/ 資料夾（09:00~13:00 各時段前10名）
           +
讀取 sim_trades.json（目前持倉中股票）
           ↓
去重合併（同一股出現在多個時段，標示所有來源）
           ↓
從 prev_scores.json 補充外資/投信資料
           ↓
下載每一檔近2年週K線（Yahoo Finance）
           ↓
計算：週MA10/MA20/MA52、近13週高低、近52週高低、週樞紐點
           ↓
判斷支撐清單、壓力清單，選出最近一層/第二層
           ↓
依現價位置決定進場策略：支撐買進 / 低位進場 / 均線支撐 / 現價觀察
           ↓
計算進場價、出場目標、停損、報酬風險比（R:R）
           ↓
產生 HTML 報告 + 寄送 Email
```

| 時段 | 執行內容 |

## 分析欄位說明

| 欄位 | 說明 | 來源 |
|------|------|------|
| 股票 / 來源 | 代號、名稱、類股 + 出現時段標籤 | hourly HTML / sim_trades |
| 現價 / 趨勢 | 收盤價 + 週線趨勢判斷 | hourly HTML |
| 週均線 | 週MA10 / 週MA20 | Yahoo Finance 週K |
| 支撐 | 最近支撐價（最多2層） | 計算值 |
| 壓力 | 最近壓力價（最多2層） | 計算值 |
| 策略 / 法人 | 進場策略說明 + 外資/投信買賣方向 | 計算值 + prev_scores |
| 進場價 | 建議進場參考價位 | 計算值 |
| 出場目標 | 壓力下方 1%（最多顯示2個目標） | 計算值 |
| 停損 | 支撐下方 3% | 計算值 |
| R:R | 報酬風險比 = (目標−進場) ÷ (進場−停損) | 計算值 |

## 支撐/壓力計算說明

### 支撐來源

| 指標 | 說明 |
|------|------|
| 週MA10 | 10週收盤移動平均 |
| 週MA20 | 20週收盤移動平均（≈半年線） |
| 週MA52 | 52週收盤移動平均（年線） |
| 近13週低點 | 過去13週最低收盤（季支撐） |
| 樞紐 S1 | = 2×樞紐點 − 上週高 |
| 樞紐 S2 | = 樞紐點 − (上週高 − 上週低) |

> 僅低於現價 1% 以內的才列為「支撐」；最近一層為「關鍵支撐」。

### 壓力來源

| 指標 | 說明 |
|------|------|
| 近13週高點 | 過去13週最高收盤（季壓力） |
| 近52週高點 | 年度最高收盤（年壓力） |
| 樞紐 P | = (上週高 + 上週低 + 上週收) ÷ 3 |
| 樞紐 R1 | = 2×樞紐點 − 上週低 |
| 樞紐 R2 | = 樞紐點 + (上週高 − 上週低) |

> 僅高於現價 1% 以上的才列為「壓力」；最近一層為「關鍵壓力」。

## 進場策略判斷邏輯

| 策略名稱 | 條件 | 進場價計算 |
|---------|------|-----------|
| **支撐買進** | 現價 ≤ 關鍵支撐 × 1.03（貼近支撐） | 支撐 × 1.005 |
| **低位進場** | 現價位於壓力支撐區間下段 < 35% | 現價 × 1.005 |
| **均線支撐** | 現價介於 週MA20 ×0.98 ~ ×1.05 | 現價 × 1.008 |
| **現價觀察** | 其餘（區間中上段） | 現價 × 1.010 |

---

## 持倉中欄位說明

持倉中的股票會額外顯示（紫色底色）：

- **進場價**：模擬進場時的價格
- **損益%**：(現價 − 進場價) ÷ 進場價 × 100%（紅=獲利，綠=虧損）
- **持有天數**：進場至今的交易日數

| 資料 | 來源 |

## Email 通知規則

與選股工具相同規則：

- **每次執行後自動寄信**給所有收件人
- **內文**：Gmail 相容 HTML，開信即見完整報告（無 JS）
- **附件**：`weekly_analysis.html`（下載後可在瀏覽器開啟）
- **主旨格式**：`[週K分析] YYYY-MM-DD HH:MM 共N檔 最佳R:R:代號名稱(X.X)`
