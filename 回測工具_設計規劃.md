# 台股選股回測工具 — 設計規劃

**腳本名稱：** `tw_stock_backtest.py`
**建立日期：** 2026-04-08
**對應主工具：** `tw_stock_screener_v2.py`（日線波段選股）

---

## 一、目標

針對現有選股工具的**日線評分邏輯 (d_score)**，在歷史資料上重播每個交易日的信號，
統計「符合條件進場後，持有 5 個交易日的整體勝率與報酬分布」，提供策略有效性的客觀驗證。

---

## 二、設計原則

- **避免未來偏差 (Look-ahead Bias)**：計算每日信號時，僅使用該日收盤前已知的歷史資料。
- **與主工具邏輯一致**：使用與 `_score_from_ind()` 完全相同的指標計算方式，確保回測結果可反映實際選股行為。
- **獨立執行**：不影響現有選股排程，手動執行。

---

## 三、現有選股評分規則（最新版確認）

### 3.1 前置過濾條件
進入評分前，必須通過以下過濾，**否則直接排除**：
- 平均日成交量 (`vma`) < **1,000,000 股（1000 張）** → 排除（低流動性）
- K 線資料筆數 < 30 根 → 排除（資料不足）

### 3.2 日線評分規則 (`d_score`)
使用參數：`D_RSI_MIN=25`、`D_RSI_MAX=60`、`D_VOL_SURGE=1.5`

| 信號 | 條件 | 加分 |
|------|------|------|
| KD黃金交叉 | 前日 K<D，當日 K>D | +25 |
| KD低檔 | K<30 且 D<30 | +10 |
| MACD柱翻正 | MACD histogram 由負轉正 | +20 |
| MACD增強 | MACD histogram > 前日且 > 0 | +8 |
| RSI適中 | RSI 在 25~60 之間 | +10 |
| RSI偏低 | RSI < 25 | -5 |
| 均線多頭 | MA5 > MA10 > MA20 | +15 |
| 均線偏多 | MA5 > MA20（不完整多頭） | +5 |
| 站上MA20 | 收盤 > MA20 | +10 |
| 量能放大 | 成交量 > 20日均量 × 1.5 | +10 |
| 布林反彈 | 前日收盤 < 布林下軌，當日突破 | +10 |
| RSI過熱 | RSI > 72 | -15 |
| 跌破MA60 | 收盤 < MA60 | -5 |
| **盤整型態** | 大量紅K後縮量整理支撐（另外函數偵測） | +8 |

分數上限：100，下限：0。

### 3.3 完整 combined 分數計算（現有系統）
```
盤中（市場開盤）：base = h_score * 0.6 + d_score * 0.4
盤後（無即時60分線）：base = d_score * 0.7 + h_score * 0.3
無60分線資料：base = d_score
combined = min(100, base + rev_score)
```
- `rev_score`：月營收 YoY 評分，最高 +20 分（>20% YoY）
- 進場門檻：`combined >= 60`

## 四、回測範圍與限制說明

### 回測可還原的項目
- 日線技術指標（KD/MACD/RSI/MA/量能/布林）
- 盤整型態偵測
- 成交量過濾（vma < 1000 張排除）

### 回測**無法**還原的項目（原因說明）
| 項目 | 無法還原原因 |
|------|------------|
| 60分線 h_score | yfinance 歷史 60 分 K 最多回溯 60 天，無法覆蓋 2 年 |
| 月營收 rev_score | FinMind 僅提供最近資料，歷史各月的值難以還原 |

### 影響評估
由於回測只含 d_score（無 rev_score 最多 +20，無 h_score 加成），
**回測的進場門檻 `d_score >= 60` 實際上比真實系統更嚴格**，
因此回測勝率若仍達標，代表純技術面已有效；真實系統加上基本面分數後，期望勝率只會更高或相近。

## 六、執行流程

### Step 1：下載歷史資料
- 使用 `yfinance` 批次下載所有上市股票過去 2 年日線 OHLCV
- 加上 `.TW` 後綴（回測僅含上市，排除上櫃以控制下載量）
- 失敗的股票跳過並記錄

### Step 2：成交量過濾
- 計算每支股票整段歷史期間的平均日量
- 平均日量 < 1,000,000 股者排除（與主工具一致）

### Step 3：信號重播（逐日掃描）
對每支股票、每個歷史交易日（排除最後 5 天，因無法確認出場）：
1. 截取「該日及之前」的 K 線資料（rolling window）
2. 計算所有指標：KD、MACD、RSI、MA5/MA10/MA20/MA60、成交量均量、布林通道
3. 套用與主工具**完全相同**的評分規則（含盤整型態偵測）
4. 若 `d_score >= 60` → 記錄為「進場信號」

### Step 4：計算持倉報酬
- **進場價**：信號日收盤價
- **出場價**：進場日後第 5 個交易日的收盤價
- **報酬率** = (出場價 - 進場價) / 進場價 × 100%
- 同一股票同一日只進場一次（避免重複計算）

### Step 5：統計彙整
彙整所有交易記錄，計算下列指標：

| 指標 | 說明 |
|------|------|
| **整體勝率** | 報酬 > 0% 的交易占比 |
| **平均報酬** | 所有交易的平均報酬率 |
| **中位數報酬** | 排除極端值干擾 |
| **最大單筆獲利 / 虧損** | 風險邊界參考 |
| **各信號組合勝率** | KD黃金交叉、MACD柱翻正、均線多頭等個別驗證 |
| **各分數區間勝率** | d_score 60-80 / 80-100 的勝率比較 |
| **年化報酬估算** | 換算整體策略效益（假設資金持續輪動） |
| **最大回撤** | 模擬連續操作下的最大虧損幅度 |

## 七、輸出報告

- **路徑：** `C:\Users\s8711\OneDrive\桌面\stock_reports\backtest\`
- **格式：** HTML（風格與現有選股報告一致）
- **檔名：** `backtest_YYYYMMDD.html`

### 報告內容
1. **總覽區塊** — 整體勝率、平均報酬、交易筆數、年化報酬估算
2. **分數區間分析** — d_score 60-80 / 80-100 的勝率與平均報酬比較
3. **信號組合分析** — 各技術信號觸發時的個別勝率排行
4. **最佳進場組合建議** — 根據回測結果，推薦勝率最高的信號組合
5. **交易明細** — 最近 50 筆回測交易記錄（含代號、進出場日、報酬率、觸發信號）
