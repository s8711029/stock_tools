## 四、如何解讀報告

報告分為**上市推薦股**（深藍色標題）與**上櫃推薦股**（深黃色標題）兩張獨立表格，各取前 10 名，互不混排。

### Step 1：先看有沒有橘色警示框（13:20 買入提醒）

若信件頂端有橘色框，優先研究這幾檔，代表爆量+紅K+分數達標（上市+上櫃皆可能出現）。

### Step 2：看各市場前幾名 + 法人欄位

- 綜合分最高的是當天各市場訊號最強的標的
- 代號旁**深藍色標籤 = 上市**，**深黃色標籤 = 上櫃**
- 外資/投信有 `**` = 法人連續 3 天以上買超，可信度較高

### Step 3：優先看黃底列（共振訊號）

**黃底 = 日線分 >= 40 且 60分線分 >= 40**，代表波段與短線同時出現訊號。

### Step 4：看盤整型態

訊號欄出現 `支撐$XXX整理N天` = 有強支撐，可以以支撐價當停損參考。

### Step 5：看月營收YoY

- 紅色正數（如 `+22%`）= 公司營收成長，技術訊號更可靠
- 綠色負數（如 `-15%`）= 公司營收衰退，即使技術分高也要謹慎

### Step 6：點代號直接看 TradingView

點擊代號開 K 線圖確認走勢（上市連結 `TWSE:代號`、上櫃連結 `TPEX:代號`）；點 TV 按鈕可自行加指標；點 K 線按鈕開 goodinfo.tw 查基本面。

## 五、Email 設定

Email 設定存放在 `stock_email_config.json`（桌面根目錄）：

```json
{
  "sender_email": "寄信帳號@gmail.com",
  "sender_app_password": "Gmail 應用程式密碼（16碼）",
  "recipient_email": [
    "收件人1@gmail.com",
    "收件人2@gmail.com"
  ]
}
```

> Gmail 應用程式密碼申請：Google 帳戶 → 安全性 → 兩步驟驗證 → 應用程式密碼

## 六、常見問題

| 問題 | 解法 |
|------|------|
| 腳本跑完說「今日休市」不執行 | 週末/假日正常，手動執行則請透過 Claude Code 強制執行 |
| 月營收顯示「—」 | FinMind 無此股資料（部分小型股），不影響技術分 |
| 60分線顯示「—」 | Yahoo Finance 無此股小時線資料（部分小型股） |
| Email 沒收到 | 確認 Gmail 應用程式密碼是否正確；Gmail 需開啟兩步驟驗證 |
| 股票清單只有 20 檔 | 確認 `lxml` 有安裝：`pip install lxml` |
| 上櫃股票清單抓取失敗 | 網路問題或 TPEX 網站暫時無回應，當次執行僅掃上市，下次自動重試 |
| 上櫃法人欄顯示「—」 | TPEX 法人 API 當日尚未更新（盤中），不影響技術分 |

腳本第 62 行開始的 `TW_HOLIDAYS` 清單需每年依行政院公告更新：

```python
TW_HOLIDAYS = {
    "2027-01-01",  # 元旦
    # ...依公告補充
}
```

兩支腳本（`tw_stock_screener_v2.py` 和 `tw_stock_disposition.py`）各有一份，都要更新。
